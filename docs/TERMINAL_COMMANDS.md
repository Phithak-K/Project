# คู่มือรวมคำสั่ง Terminal สำหรับระบบ SwiftPath (Terminal Commands Cheatsheet)

เอกสารฉบับนี้รวบรวมคำสั่งยอดนิยมที่ต้องใช้งานในเทอร์มินัล (Terminal / PowerShell / Command Prompt) ในการควบคุม รันระบบ และดูแลรักษาโครงสร้างโปรเจกต์ SwiftPath ทั้งหมด

---

## 📁 1. การนำทางพื้นฐาน (Navigation)
ก่อนเริ่มใช้คำสั่งใด ๆ คุณต้องแน่ใจว่าเปิดหน้าต่างเทอร์มินัลอยู่ในโฟลเดอร์ที่ถูกต้อง:
* **ไปยังโฟลเดอร์โครงการหลัก (Root Folder):**
  ```bash
  cd d:\Project
  ```
* **ไปยังโฟลเดอร์ Backend:**
  ```bash
  cd d:\Project\backend
  ```
* **ไปยังโฟลเดอร์ Frontend:**
  ```bash
  cd d:\Project\frontend
  ```

---

## 🗄️ 2. คำสั่งควบคุมฐานข้อมูล (Docker Database)
*รันคำสั่งเหล่านี้เมื่ออยู่ที่ **Root Folder** (`d:\Project`):*

| คำสั่ง | คำอธิบายการใช้งาน |
| :--- | :--- |
| `docker compose up -d` | **เริ่มรันฐานข้อมูล** PostgreSQL ในเบื้องหลัง (Background) |
| `docker compose down` | **หยุดรันและลบตู้คอนเทนเนอร์** ฐานข้อมูลออกเมื่อเลิกใช้งาน |
| `docker compose restart` | **รีสตาร์ทฐานข้อมูล** (ใช้เมื่อเชื่อมต่อหลุดหรือไม่ยอมตอบสนอง) |
| `docker ps` | **ตรวจสอบสถานะคอนเทนเนอร์** เพื่อดูว่าพอร์ต `5432` รันอยู่ปกติหรือไม่ |
| `docker logs -f logistics_db` | **เปิดดูบันทึก (Logs)** ของตู้ฐานข้อมูลแบบสด ๆ (กด `Ctrl + C` เพื่อออก) |
| `docker exec -it logistics_db psql -U phithak -d logistics_v1` | **ล็อกอินเข้าหน้าควบคุม SQL** ของฐานข้อมูลผ่านแอป Terminal โดยตรง |

---

## ⚙️ 3. คำสั่งจัดการฝั่งเบื้องหลัง (NestJS Backend)
*รันคำสั่งเหล่านี้เมื่ออยู่ที่โฟลเดอร์ **`backend`** (`d:\Project\backend`):*

| คำสั่ง | คำอธิบายการใช้งาน |
| :--- | :--- |
| `npm install` | **ติดตั้งโปรแกรมเสริม (Libraries)** ทั้งหมดที่โครงการเรียกใช้งาน (รันครั้งแรก) |
| `npm run start:dev` | **รันระบบทดสอบ (Development)** ซึ่งจะทำการโหลดระบบใหม่ทันทีเมื่อเซฟไฟล์โค้ด |
| `npm run build` | **แปลโค้ดระบบ (Build)** ให้กลายเป็นโฟลเดอร์ `dist` สำหรับใช้งานจริง |
| `npm run start:prod` | **รันระบบจำลองใช้งานจริง** (ต้องผ่านการ build ก่อนเสมอ) |

---

## 💎 4. คำสั่งจัดการข้อมูลด้วย Prisma (Prisma ORM)
*รันคำสั่งเหล่านี้เมื่ออยู่ที่โฟลเดอร์ **`backend`** (`d:\Project\backend`):*

| คำสั่ง | คำอธิบายการใช้งาน |
| :--- | :--- |
| `npx prisma db push` | **ซิงค์โครงสร้างตาราง** จาก `schema.prisma` เข้าไปยัง Database โดยตรงทันที |
| `npx prisma studio` | **เปิดเว็บหน้าต่างจำลองฐานข้อมูล (GUI)** ช่วยให้กดดูตาราง เพิ่ม ลบ แก้ไขข้อมูลได้บนเบราว์เซอร์ |
| `npx prisma generate` | **สร้าง Prisma Client ใหม่** เพื่อซิงค์ชนิดข้อมูลในฐานข้อมูลให้เข้ากับ Typescript |

---

## 💻 5. คำสั่งจัดการฝั่งหน้าเว็บ (Next.js Frontend)
*รันคำสั่งเหล่านี้เมื่ออยู่ที่โฟลเดอร์ **`frontend`** (`d:\Project\frontend`):*

| คำสั่ง | คำอธิบายการใช้งาน |
| :--- | :--- |
| `npm install` | **ติดตั้งโปรแกรมเสริม** ฝั่งหน้าบ้าน (รันครั้งแรก) |
| `npm run dev` | **รันเซิร์ฟเวอร์ทดสอบหน้าบ้าน** บนพอร์ตหลัก `http://localhost:3000` |
| `npm run build` | **Build หน้าเว็บทั้งหมด** ให้พร้อมสำหรับนำขึ้นเซิร์ฟเวอร์จริง (Production) |
| `npm start` | **รันเว็บที่สร้างแล้ว (Production Mode)** บนพอร์ต `3000` |

---

## 💡 ทิปส์น่ารู้ในการใช้งาน (Tips & Tricks)
1. **การปิดระบบที่กำลังทำงาน:** หากต้องการหยุดเซิร์ฟเวอร์ที่กำลังรันอยู่ (เช่นหลังรัน `npm run dev` หรือ `npm run start:dev`) ให้กดคีย์ลัด `Ctrl + C` แล้วกด `Y` และ `Enter` ใน Terminal
2. **หากระบบค้างหรือรันไม่ขึ้น:**
   * ให้ลองลบโฟลเดอร์เก็บข้อมูลชั่วคราวแล้วดาวน์โหลดใหม่:
     ```bash
     # สำหรับ Backend
     rm -rf node_modules dist
     npm install
     
     # สำหรับ Frontend
     rm -rf node_modules .next
     npm install
     ```
